The Goal

Score as many points as possible.
Output a single line as a solution.
Warning: the validation tests used to compute the final score are not the same as the ones used during the event. Hardcoded solutions will not score highly.

  Rules

The game is played on a platform in space. Due to Magus' poor AI skills Automaton2000 only walks in straight lines. Guide Automaton2000 with arrows to save him from stepping off the platform!
Unfortunately, due to Magus' poor coding Automaton2000 also crashes if he revisits a state he's been in, as he realises he's going in circles!
Try to keep Automaton2000 alive as long as possible, you score 1 point per turn that he is.

The game works as follows:
  • The platform is a grid of height 10 and width 19
  • You are given the layout of the platform in ascii form with #:void, .:platform, U:Up arrow, R:Right arrow, D:Down arrow, L:Left arrow
  • Place arrows on your first turn by outputting in the format: X Y U/R/D/L
  • Automaton2000 moves in a straight line and only changes direction when stepping on an arrow
You may not place arrows on top of existing ones.

  Expert Rules

Order of events in a turn:
  • Score is incremented by 1 for each Automaton2000 alive
  • Automaton2000 moves by 1 cell in the direction he's facing
  • Automaton2000 changes direction if he's located on an arrow
  • Automaton2000 dies if he has stepped into the void or entered a (position,direction) state he has been in before
For the first turn Automaton2000 changes direction if he's standing on an arrow (i.e: You can change the initial direction of a robot by placing an arrow under it). You can run custom test cases in the IDE. You can access the referee code on a public github.

  Note

Don’t forget to run the tests by launching them from the “Test cases” window. You can submit at any time to receive a score against the training validators. There are some limitations on submits per unit time. Your most recent submission will be used for the final ranking.

Don't hesitate to change the viewer's options to help debug your code ().

  Game Input

Input

19 strings of 10 characters each representing #:void, .:platform, U:Up arrow, R:Right arrow, D:Down arrow, L:Left arrow

1 line containing the number of robots

1 line for each robot of the form X Y DIR with X Y the initial position of the robot and DIR the direction it faces (U, R, D or L).

Output
A single line containing triplets of the form X Y DIR with X Y two integers and DIR a character, either U,R,D or L.
Constraints
1≤robots≤18.

Allotted response time to output is ≤ 1 second.
